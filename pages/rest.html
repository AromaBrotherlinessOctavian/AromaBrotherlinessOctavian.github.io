REST архитектура

В последнее время, большинство Web-API использует архитектуру REST с представлением данных в виде JSON документов.

Пользователь делает запрос на получение ресурса, который идентифицируется URL, и получает информацию о ресурсе в виде JSON.

REST - это архитектурный стиль, в основе которого лежат принципы:

  • Однообразности интерфейса вся API в целом имеет однородный стиль
  • Архитектура Клиент-Сервер
  • Отсутствие состояния все нужные данные содержатся в сообщениях, и сервер ничего не "запоминает"
  • Кеширование клиент может кешировать представления ресурсов, когда это имеет смысл
  • Многослойность клиент общается только с API, а что стоит дальше за ней стоит, ему неизвестно и не интересно

Оценка REST

Достоинства Недостатки
Простота Отсутствие состояния - иногда имеет смысл хранить состояние на сервере, а не постоянно таскать его с собой на клиенте
Гибкость Иногда строгость и контроль лучше, чем гибкость
Популярность Использование HTTP и архитектуры запрос-ответ (иногда нужна прямая двухсторонняя коммуникация)
