Использование API сервисов при разработке сайтов

Как выбирать Web-API

Критерии выбора API

Критерий Тип критерия Описание Вес
Соответствие задаче Функциональные Возможность осуществления поставленной задачи с помощью данного функционала 5
Гибкость интеграции Функциональные Возможность легко встроить API в нужную систему 5
Обширность Функциональные Возможность продолжить использовать ту же API в будущем, при расширении задачи 3
Производительность Технические Быстрота реализации функционала 4
Надёжность Технические Доступность API влияет и на доступность самого приложения 5
Безопасность Технические Данные, которые сообщаются API и которые получаются от API, должны быть защищены от злоумышленников 5
Сообщения об ошибках Технические Если начинаются проблемы, то грамотные сообщения об ошибке помогают быстрее их устранять 1
Качество документации Удобство разработки Если пользователь не сможет найти что-то в документации, ему придётся разбираться самому, лезть в исходный код (где возможно) и тратить время 3
Стабильность API Удобство разработки Изменения API в идеале должны быть обратно-совместимы или использовать версионирование, чтобы не пришлось переписывать продукты, использующие эту API 2
Уровень абстракции Удобство разработки Чем абстрактнее API и чем меньше она зависит от конкретных обстоятельств и технологий, тем легче интегрировать её в различные системы 2
Лицензионные критерии выбора API Легальные Так как API - чей-то продукт, то и его использование разрешается в рамках некого соглашения с владельцем/оператором данного продукта. Иногда эти соглашения мешают выполнению поставленной задачи 2
Лимиты Легальные Иногда владельцы/операторы API накладывают ограничения на пользователей (например, лимиты трафика), что может мешать выполнению поставленной задачи 3
Гарантии Легальные Некоторые владельцы/операторы API могут давать некие гарантии по праву пользования API, но если этих гарантий нет, то владелец/оператор может отозвать право на использование API в любой момент. 2
