Как выбирать Web-API

Критерии выбора API

Критерий Описание Вес
Функциональные критерии выбора API
Соответствие задаче Возможность осуществления поставленной задачи с помощью данного функционала 5
Гибкость интеграции Возможность легко встроить API в нужную систему 5
Обширность Возможность продолжить использовать ту же API в будущем, при расширении задачи 3
Технические критерии выбора API
Производительность Быстрота реализации функционала 4
Надёжность Доступность API влияет и на доступность самого приложения 5
Безопасность Данные, которые сообщаются API и которые получаются от API, должны быть защищены от злоумышленников 5
Сообщения об ошибках Если начинаются проблемы, то грамотные сообщения об ошибке помогают быстрее их устранять 1
Критерии выбора API по удобству разработки
Качество документации Если пользователь не сможет найти что-то в документации, ему придётся разбираться самому, лезть в исходный код (где возможно) и тратить время 3
Стабильность API Изменения API в идеале должны быть обратно-совместимы или использовать версионирование, чтобы не пришлось переписывать продукты, использующие эту API 2
Уровень абстракции Чем абстрактнее API и чем меньше она зависит от конкретных обстоятельств и технологий, тем легче интегрировать её в различные системы 2
Критерии выбора API по удобству разработки
Лицензионные критерии выбора API Так как API - чей-то продукт, то и его использование разрешается в рамках некого соглашения с владельцем/оператором данного продукта. Иногда эти соглашения мешают выполнению поставленной задачи 2
Лимиты Иногда владельцы/операторы API накладывают ограничения на пользователей (например, лимиты трафика), что может мешать выполнению поставленной задачи 3
Гарантии Некоторые владельцы/операторы API могут давать некие гарантии по праву пользования API, но если этих гарантий нет, то владелец/оператор может отозвать право на использование API в любой момент. 2
